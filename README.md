# ActiumDigital Organization Configuration

Central repository for organization-wide GitHub configuration used across ActiumDigital projects.

## Documentation Index

| Folder | Purpose | Documentation |
| --- | --- | --- |
| `.github/workflows/` | Team automation workflows | [`.github/workflows/README.md`](./.github/workflows/README.md) |
| `workflows/` | Release and changelog workflows | [`workflows/README.md`](./workflows/README.md) |
| `ISSUE_TEMPLATE/` | Issue templates and issue form config | [`ISSUE_TEMPLATE/README.md`](./ISSUE_TEMPLATE/README.md) |
| `teams/` | Teams-as-code configuration | [`teams/README.md`](./teams/README.md) |
| `templates/` | Reusable project templates | [`templates/README.md`](./templates/README.md) |

## Root Files

- [`CONVENTIONAL_COMMITS.md`](./CONVENTIONAL_COMMITS.md) — Commit message standard for automated versioning.
- [`CHANGELOG_TEMPLATE.md`](./CHANGELOG_TEMPLATE.md) — Changelog starter template for repositories.

## Quick Start

1. Configure teams in [`teams/teams.json`](./teams/teams.json).
2. Use issue templates from [`ISSUE_TEMPLATE/`](./ISSUE_TEMPLATE/).
3. Copy project scaffolding from [`templates/`](./templates/).
4. Follow [Conventional Commits](./CONVENTIONAL_COMMITS.md) to drive versioning workflows.
